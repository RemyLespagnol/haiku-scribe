# v0.2 — Credibility & Adoption Pass (design)

Date: 2026-07-09
Status: approved (brainstorming)

## Goal

Make the *existing* plugin land and be believed. No new shipped surface, no
maintained proof machinery. This is a docs + packaging pass, targeting release
`v0.2`.

Chosen axes (from brainstorming): **adoption/diffusion** + **credibility**, with
proof taken as *explanation only* — the heavy benchmark deleted at the plugin
pivot stays deleted.

## Non-goals

- No reproducible benchmark / no measured gain numbers presented as data.
- No external discoverability work this round (awesome-lists, directories,
  GitHub topics) — deferred.
- No change to the three shipped surfaces' behavior (`agents/`, `hooks/`,
  manifests). The contract and its test are untouched.

## Deliverables

### 1. Illustrative walkthrough in README

New section between **Why** and **Install**. A concrete *without / with* on one
representative task (e.g. "map how auth flows across ~8 files"):

- **Without**: the main model reads the 8 files itself → ~thousands of tokens of
  raw source sit in the main context for the rest of the task.
- **With**: it delegates → the scout returns a ~40-line structured brief → the
  main context holds only that.

Framed explicitly as *illustrative* ("typical shape", not a benchmarked figure)
to honor the explanation-only decision. Includes a real sample of the scout's
`Response Shape` output (Summary / Evidence / Unknowns / Suggested Direct Reads)
so the reader sees what they get back. This section *is* the "proof by
explanation".

### 2. Demo asset — VHS `.tape` → GIF

- Source: `assets/demo.tape` (charmbracelet/vhs script, versioned plain text).
- Output: `assets/demo.gif`, embedded near the top of the README.
- The `.tape` scripts a short terminal demo of the auto-route → brief flow.
- Rationale for VHS over Remotion: VHS is a single external binary driving a
  text script — no Node/Chromium/ffmpeg pipeline added to a repo that today has
  zero build. Remotion's video toolchain contradicts the project's
  zero-machinery ADN. Static SVG was the zero-tool alternative but GitHub does
  not animate SVG embedded as `<img>`, so animation requires a GIF.
- The committed `.gif` is a binary; regeneration needs `vhs` installed locally.
  Document the regen command near the tape file or in README Development.

### 4. CI workflow

`.github/workflows/ci.yml`, minimal:

- Always: `python3 test_contract.py` (and/or `python -m pytest -q`) + `ruff check`.
- Best-effort: `claude plugin validate .`.
  - **Risk**: the `claude` CLI may not be installable/authable in a GitHub
    Actions runner. If it is not cleanly available, fall back to a lightweight
    JSON validity check of `.claude-plugin/plugin.json` and
    `.claude-plugin/marketplace.json` (e.g. `python -c "import json; ..."`), so
    the badge still reflects a real green signal. Decide at implementation time
    based on what actually runs; do not block the workflow on an unavailable CLI.
- This workflow powers the CI badge (deliverable 3) and is the cheapest real
  credibility signal — zero narrative upkeep.
